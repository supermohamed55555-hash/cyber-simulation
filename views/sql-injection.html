Educational Breakdown

What Happened?

The input ' OR '1'='1 was appended directly to the SQL query. This changed the logic of the search from "find a user with this specific name" to "find any record where the name matches OR 1=1 is true". Since 1=1 is always true, the database returned every row.

Why it Worked?

  • Lack of Input Sanitization: The application didn't check for special SQL characters like single quotes (').
  • Dynamic Query Building: The query was built using simple string concatenation: "SELECT ... WHERE name = '" + input + "'".

How to Prevent it?

  • Prepared Statements: Use parameterized queries where the database treats input as data ONLY, never as executable code.
  • Input Validation: Only allow expected characters (e.g., alphanumeric only for usernames).
  • Principle of Least Privilege: Ensure the web application's database user can only access the tables it absolutely needs.
